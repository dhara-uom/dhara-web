Apache Rave

A web portal is often the front end component of a gateway. It presents the various capabilities of gateway in a user-friendly and uniform manner. The portal manages user accounts and users are allowed to select and use required capabilities of the gateway. In order to demonstrate a portal for our solution, we use Apache Rave.

Apache Rave is a web and social mashup engine and a light weight platform which serves web widgets [54]. It integrates Apache Shindig & Apache Wookie to provide OpenSocial & W3C widget support, respectively. Apache Rave is another Open Source software used in our project. It provides strong context-aware personalization, collaboration and content integration capabilities and a high quality out-of-the-box installation as well as be easy to integrate in other platforms and solutions.

Main features of Apache Rave are as follows:

  • Personalized content (for each user)
  • Shared web content (between users)
  • Customizable
  • Multiplatform (Mobile and Desktop View)
  • Available widget store
  • Admin interface for users and widgets
  • User and Group management
  • Pluggable security/authentication
  • Inter-Widget Communication (IWC)

The main feature of Apache Rave that has been of most importance in our project is its capability to be extended into a customized portal based on user requirements. It allows the custom portal developers to use custom components as well as to customize look and feel. Apache Rave is built using Spring MVC framework which makes extension easier and simpler.
