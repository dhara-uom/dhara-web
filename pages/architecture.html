High-level Design

Dhara is a geoscience gateway which targeting on facilitating a user friendly and less complex environment for geoscientist to perform their experiments. As detailed previous about Apache Airavata which is capable of create and run workflows using HPC resources perform a key role in Dhara portal. Overall layered system architecture is shown in Figure 3 Dhara portal is a Spring MVC based web application which builds using Apache Rave. Apache Rave is a framework for build science gateways. One of interested featured of Apache Rave is the function of widgets. Developer can develop custom widgets which can be a new feature or a service for the portal. This widget can be added to the portal and then users can select which widgets need to be in their user accounts. Extending Apache Rave to build the Dhara portal give this added advantage along with features coming as defaults such as user account management.

According to the layered architecture the top layer is the UI layer where end users interact with the portal. View layer interact with controllers to give the correct view for users. Dhara portal follows the MVC architecture which maintains the relationship with model, view and controller. Each page of portal is communicating and map with a controller to give the required functions to the user.

Service layer gives the services to perform portal functions. Dhara uses Spring beans to implement the services which can easily use by other classes or controllers to achieve their target functionality. Dhara service layer has four major service components. Apache Airavata API is exposed as a core service in the service layer. Airavata service allows retrieving, executing and monitoring workflows and other details from portal. Code generation service use to perform the code generation for user desired workflow and 52 north service is responsible for deploying generated class for a workflow in 52o WPS instance. Other services are fall under portal services.

Data access layer provides data exchange between database servers and portal. Dhara uses Hibernate framework for data persistence and retrieving activities which gives high degree of standardization within data access layer.
